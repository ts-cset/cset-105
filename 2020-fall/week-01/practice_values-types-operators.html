Practice Problems: Values, Types, and Operators

One of the best ways to practice programming is using the scientific method. Look at a line of code, come up with your guess at what it will do, then run it to see if you were correct.

Here are a list of JavaScript expressions using the values and operators in Chapter 1. See if you can figure out what the answer will be, then run the code in the book's Code Sandbox, like this: 


// Run code here in the context of Chapter 1
console.log(1 + 2);

You don't need to know what console.log() means, just know that it will print out the result of the expression inside the parentheses. You will, and should, use it a lot in your programs to "see" what a value or variable is. Run the code by hitting the "run code" button, or pressing Command + Enter (Control on Windows).

Arithmetic

  • 5 + 10 * 3
  • (1 + 2) * (3 + 1)
  • 1 + 2 * 3 + 1
  • 25 + 30 / 6
  • 8 % 3
  • 5 ** 2
  • -(-1 - 1)

Strings

  • 'Hello' + 'username'
  • "I can't be surrounded with single quotes"
  • 'I can\'t unless I \'escape\' the inner single quotes'
  • `I am ${5.85 * 12} inches tall`
  • typeof "13"

Booleans

  • 3 < 5
  • 3 + 2 < 5 - 7
  • 5 > -2
  • (5 >= 5)

Common Questions

Why does the Code Sandbox not give me the right answer? Remember, computers are stubbornly stupid. Make sure that you've typed every single character as exactly as you wanted, even the ones you can't see like spaces and new lines with enter. If that doesn't work, show your code to a classmate and see if they get the same answer.

Wait, the ** operator wasn't in the book, was it? Nope. There are a lot more and we'll learn them as we go. If you ran it and got a result, take a guess at what it does. Or you can search the internet for an answer.
